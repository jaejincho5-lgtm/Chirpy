## 1. What changed and why

### /voice (app/voice/page.tsx + app/globals.css)
- **Caption bubble hidden by default.** New "Phụ đề" pill toggle in the header (`.voice-cc`).
- **Error paths force captions on** (mic denied ×2, speech unsupported, recognition error streak,
  expired Chirpy link) so instructions are never invisible while captions are off.
- **Tap the chicken → repeats the last meaningful line** (greeting or assistant reply — never the
  "Dạ, để em xem…" fillers). Keyboard accessible (Enter/Space on the avatar). Blocked while the
  agent is busy; cancels current audio first; a `speechTokenRef` guard stops rapid double-taps from
  leaving `speaking` stuck. Mic auto-pauses during the repeat via the existing echo-guard effect.
- All speech now goes through one `speakLine(text, {meaningful, cancelFirst})` helper.

### Font fix ("Telex text looks out of place" in backend)
- Root cause: **Spline Sans Mono has no Vietnamese subset** — diacritics in mono-styled text
  (KPI values, receipts, trace rows) fell back to another face mid-word. No config can fix that.
- **Swapped mono to IBM Plex Mono** (`app/layout.tsx`) — full Vietnamese coverage, same weights,
  same `--font-mono` variable, nothing downstream changed.
- Two spots rendering Vietnamese *sentences* in mono moved to the UI face:
  `.transcript__busy` and `.trace-console__empty` (globals.css). Numbers/codes stay mono.

### Bugs found + fixed (each reproduced by a test first)
1. **`lib/say.ts`: chicken could speak "json" aloud.** A reply truncated mid-```` ```json ```` fence
   made `extractSay` return the literal fence marker. Added `salvageSay`/`stripFenceArtifacts` —
   never speaks fence markers or raw JSON; recovers the partial `say` value when possible.
   Covered by `tests/speech-text.test.ts` (24 cases: emoji stripping, truncation, escapes…).
2. **`lib/worldstate.ts`: timezone double-shift.** The +7h shift assumed a UTC host, so on a UTC+7
   laptop 11:30 read as 18:30 → wrong daypart suggestions on any localhost demo. Now correct on
   any host TZ.
3. **`db/schema.sql` said `convo_id`; live table + code use `convo_key`.** Fixed the schema file.
   ⚠️ The live DB was checked first — do NOT "fix" lib/followup.ts to `convo_id`, that would break prod.
4. **Unstyled "Chạy mô phỏng" button** in Tái kích hoạt (`.promo-submit` had no CSS rule) — styled.
5. Docs: README no longer claims the model is Opus (code default is `openai/gpt-4o`, see lib/ai.ts);
   `.env.example` now documents TWILIO_*, CHEAP_MODEL, OOS_DEMO_ITEMS.
